User Interface of 3 Applications

Reflection Questions

In each application, when the mouse hovers above the dropdown component or when the Tab key selects that component, the component be highlighted to show that the component is selectable. Some difference are noticeable like for Google Drive, when I hovered over the component there is a text that pops up, indicating the functionality of the component. This was not implemented for YouTube or Unity.

The component supports various interactive functionalities, including opening and closing via clicking the dropdown button, pressing Space or Enter, clicking outside the button, or pressing Esc. Navigation is possible using up/down arrow keys, scrolling, hovering to highlight options, and using Tab to move through items. Selection can be done by left-clicking, right-clicking for more options, or pressing Enter. However, the component does not support drag-and-drop, selecting partial text, copy-pasting, or adding text.

The component is learnable through hover states that indicate selectability, intuitive UI with clear headers, and a scrollbar for additional options. It is memorable due to frequent interaction across various applications and predictable functionality that allows users to navigate instinctively. In terms of efficiency, it requires only a simple click to open and close, eliminates the need for page relocation, and uses logical sorting to help users quickly find the desired option.
